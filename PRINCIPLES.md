# หลักการ — AI Agent Fundamentals

หลักการเหล่านี้เป็นแนวทางในการสร้างเนื้อหาและตัวอย่างในที่เก็บนี้
เพื่อให้ผู้เรียนไทยได้รับความรู้ที่มีคุณภาพ ใช้งานได้จริง และปลอดภัย

---

## 1. Thai-first

เนื้อหาหลักเป็นภาษาไทย อธิบายแนวคิดที่ซับซ้อนด้วยภาษาที่เข้าใจง่าย
ใช้ศัพท์เทคนิคภาษาอังกฤษเมื่อจำเป็น พร้อมคำอธิบายภาษาไทยกำกับทุกครั้ง

## 2. Human-in-the-loop

มนุษย์ต้องอยู่ในวงควบคุมเสมอ Agent ไม่ควรตัดสินใจสำคัญโดยไร้มนุษย์ตรวจสอบ
เราสอนให้ออกแบบระบบที่มนุษย์เป็นผู้ตัดสินใจขั้นสุดท้าย

## 3. Transparent Workflow

Agent ต้องอธิบายได้ว่ากำลังทำอะไร ทำไมถึงทำ และได้ผลลัพธ์อย่างไร
ความโปร่งใสช่วยให้ไว้วางใจและแก้ปัญหาได้เมื่อเกิดข้อผิดพลาด

## 4. Tool Safety

การให้ Agent ใช้เครื่องมือภายนอกต้องมีขอบเขตที่ชัดเจน
เราสอนแนวทาง: อนุมัติก่อนทำงานสำคัญ จำกัดสิทธิ์เท่าที่จำเป็น และตรวจสอบผลลัพธ์

## 5. Verification Matters

ทุกคำตอบและทุกการกระทำของ Agent ต้องสามารถตรวจสอบได้
การประเมินและการตรวจสอบไม่ใช่ขั้นตอนเสริม — เป็นส่วนหนึ่งของกระบวนการ

## 6. Local-first When Possible

เลือกใช้โมเดลและเครื่องมือที่ทำงานในเครื่องก่อน เมื่อเหมาะสม
ลดการพึ่งพาบริการภายนอก เพิ่มความเป็นส่วนตัวและลดความหน่วง

## 7. Composable System

ออกแบบ Agent เป็นส่วนประกอบที่นำมาต่อกันได้
แทนที่จะเป็นระบบขนาดใหญ่ที่แก้ไขยาก ใช้หน่วยย่อยที่ทดสอบและบำรุงรักษาสะดวก

## 8. Practical Over Hype

เนื้อหาต้องใช้งานได้จริง ไม่ใช่แค่แนวคิดที่สวยหรู
ทุกบทต้องมีตัวอย่างที่ผู้อ่านสามารถนำไปปรับใช้ หรืออย่างน้อยเข้าใจกลไกการทำงาน

---

## การตัดสินใจ

เมื่อเนื้อหาหรือตัวอย่างขัดแย้งกัน ให้ยึดหลักการเหล่านี้ตามลำดับ:

1. ความปลอดภัยของผู้ใช้และข้อมูล (Human-in-the-loop + Tool Safety)
2. ความถูกต้องและตรวจสอบได้ (Verification Matters)
3. ความเป็นประโยชน์ต่อผู้เรียนไทย (Thai-first + Practical)
4. ความยืดหยุ่นและบำรุงรักษา (Composable + Local-first)
5. ความโปร่งใส (Transparent Workflow)
