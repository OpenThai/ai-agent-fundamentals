# บทที่ 2: Chatbot vs AI Agent

---

ก่อนจะสร้าง AI Agent เราต้องเข้าใจก่อนว่า **อะไรคือ chatbot และอะไรคือ Agent** และที่สำคัญ — **เมื่อไรควรใช้อะไร**

---

## เปรียบเทียบแบบตาราง

| มิติ | Chatbot | AI Agent |
|------|---------|----------|
| **วิธีทำงาน** | ตอบตาม input ที่ได้รับ | วางแผนและทำงานหลายขั้นตอนเอง |
| **ความจำ** | มักจำเฉพาะรอบสนทนา | มีทั้ง short-term และ long-term memory |
| **เครื่องมือ** | ไม่มี หรือมีจำกัด | สามารถเรียกใช้เครื่องมือภายนอก |
| **การวางแผน** | ไม่มี — ตอบทีละคำถาม | มี — แตกงานและเรียงลำดับขั้นตอน |
| **การตรวจสอบตัวเอง** | ไม่มี | มี reflection และ self-correction |
| **Human-in-the-loop** | ไม่จำเป็น | ออกแบบให้มนุษย์อยู่ใน loop |
| **ความซับซ้อน** | ต่ำ | ปานกลางถึงสูง |
| **ค่าใช้จ่าย** | ต่ำ | สูงกว่า (หลายรอบ, หลาย token) |
| **ความสามารถ** | ตอบคำถาม สนทนา | ทำงานให้สำเร็จ(end-to-end) |

---

## เมื่อไรควรใช้ Chatbot

Chatbot เหมาะกับงานที่:

- **ถาม-ตอบความรู้** — เช่น "ประชากรไทยเท่าไหร่", "Python tuple กับ list ต่างกันยังไง"
- **สนับสนุนลูกค้าขั้นต้น** — เช่น เปลี่ยนรหัสผ่าน ดูยอดเงิน เช็คสถานะออเดอร์
- **ให้ข้อมูลคงที่** — เช่น นโยบายบริษัท เวลาเปิด-ปิด
- **พูดคุยทั่วไป** — ไม่ต้องการผลลัพธ์ที่ซับซ้อน

ข้อดี: ถูก เร็ว เรียบง่าย ควบคุมง่าย

### ตัวอย่าง
```
คุณ: สินค้าที่สั่งไปถึงยัง
Chatbot: ใช้เลข tracking RX123456789TH
         สถานะ: อยู่ระหว่างขนส่ง
         กะว่าถึง 25 พ.ค. นี้ครับ
```

---

## เมื่อไรควรใช้ AI Agent

Agent เหมาะกับงานที่:

- **ต้องใช้หลายเครื่องมือร่วมกัน** — เช่น ค้นข้อมูล + วิเคราะห์ + สร้างรายงาน
- **ต้องวางแผนหลายขั้นตอน** — เช่น "หาลูกค้ากลุ่มเป้าหมาย แล้วส่งอีเมล personalize"
- **ต้องปรับเปลี่ยนตามสถานการณ์** — เช่น ถ้า API ตัวแรกล้ม ให้ลองตัวที่สอง
- **ต้องตรวจสอบตัวเอง** — เช่น เขียนโค้ดแล้วรัน test ถ้าผิดให้แก้
- **ต้องจำข้อมูลข้าม session** — เช่น "เมื่อกี้เราคุยเรื่องโปรเจกต์ A ไว้ มาทำต่อ"

ข้อเสีย: แพงกว่า ช้ากว่า ซับซ้อนกว่า เจอ error ได้มากกว่า

### ตัวอย่าง
```
คุณ: วิเคราะห์คู่แข่งบริษัท A ให้หน่อย
Agent: [วางแผน] ค้นหาข้อมูล -> อ่านเว็บ -> สรุป -> สร้างตารางเปรียบเทียบ
       [ค้นหา] ข้อมูลบริษัท A ล่าสุด
       [ค้นหา] รีวิวจากลูกค้า
       [วิเคราะห์] จุดแข็ง จุดอ่อน
       [สรุป] ตารางเปรียบเทียบ + ข้อเสนอแนะ
```

---

## ไกด์ไลน์การเลือก

```
งานนี้...
├─ แค่ถาม-ตอบข้อมูลตายตัว? → Chatbot
├─ ต้องการทำงานหลายขั้นตอน? → Agent
├─ ต้องใช้ข้อมูลภายนอก? → Agent (หรือ RAG)
├─ ต้องการความถูกต้องสูง? → Agent + Verification
├─ ต้องปรับเปลี่ยนตามบริบท? → Agent
├─ ต้องประหยัดค่าใช้จ่าย? → ลอง Chatbot ก่อน
└─ เริ่มต้นไม่แน่ใจ? → Chatbot แล้วค่อยอัปเกรดเป็น Agent
```

---

## ความเข้าใจผิดที่พบบ่อย

> "Chatbot ไม่มีประโยชน์ถ้ามี Agent แล้ว"

Chatbot ยังมีประโยชน์มากสำหรับงานที่ตรงไปตรงมา ใช้ Agent ก็เหมือนจ้างผู้จัดการมาทำงานที่แค่รับโทรศัพท์ — เกินจำเป็นและสิ้นเปลือง

> "Agent ดีกว่า chatbot ทุกเรื่อง"

Agent เก่งในวงกว้าง แต่ chatbot เก่งกว่าในเรื่อง: ความเร็ว, ต้นทุน, ความแน่นอน, การควบคุม

---

## สรุป

- Chatbot เหมาะกับงานถาม-ตอบ ไม่ซับซ้อน ไม่ต้องใช้เครื่องมือ
- Agent เหมาะกับงานที่ต้องวางแผน ใช้เครื่องมือ และปรับตัว
- เลือกให้เหมาะสมกับงาน — ไม่จำเป็นต้องใช้ Agent ทุกอย่าง
- แนวทางปลอดภัย: เริ่มจาก chatbot แล้วเพิ่มความสามารถเมื่อจำเป็น
