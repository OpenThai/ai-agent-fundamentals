# คำศัพท์ — AI Agent Fundamentals

คำอธิบายศัพท์สำคัญในโลก AI Agent เป็นภาษาไทย

---

| ศัพท์ | คำอธิบาย |
|-------|----------|
| **AI Agent** | ระบบที่ใช้โมเดลภาษา (LLM) เป็นสมองหลัก สามารถวางแผน ใช้เครื่องมือ จดจำบริบท และทำงานแบบอัตโนมัติหลายขั้นตอน ต่างจาก chatbot ทั่วไปที่ตอบคำถามอย่างเดียว |
| **Chatbot** | โปรแกรมสนทนาที่ตอบคำถามตาม input ที่ได้รับ มักไม่มีหน่วยความจำต่อเนื่องหรือความสามารถในการวางแผน ไม่ได้ถูกออกแบบให้ทำงานหลายขั้นตอนด้วยตัวเอง |
| **Agentic Workflow** | รูปแบบการทำงานที่ Agent วางแผน → ลงมือ → สังเกตผลลัพธ์ → ปรับแผน → ทำซ้ำ เป็นวงจร จนกว่างานจะสำเร็จ |
| **Tool-use** | ความสามารถของ Agent ในการเรียกใช้เครื่องมือภายนอก เช่น ค้นเว็บ คำนวณ อ่านไฟล์ เรียก API โดย Agent เป็นผู้ตัดสินใจว่าควรใช้เครื่องมือใดและเมื่อไร |
| **Function Calling** | กลไกที่ LLM สร้างคำขอเรียกฟังก์ชันในรูปแบบ structured output (เช่น JSON) จากนั้นระบบจะเรียกฟังก์ชันนั้นและส่งผลลัพธ์กลับให้ LLM |
| **Memory** | ระบบที่ Agent ใช้เก็บและเรียกค้นข้อมูลข้ามการสนทนา มีทั้งแบบ short-term (จำภายในรอบเดียวกัน) และ long-term (จำข้ามเซสชัน) |
| **Context** | ข้อมูลบริบทที่ Agent มีในขณะทำงาน รวมถึงประวัติการสนทนา คำสั่ง ข้อมูลผู้ใช้ และผลลัพธ์จากเครื่องมือต่างๆ |
| **Context Window** | ขนาดสูงสุดของ token ที่โมเดลภาษาสามารถรับได้ในหนึ่งครั้ง ยิ่ง windows ใหญ่ ยิ่งจำข้อมูลได้มาก แต่ก็ใช้ทรัพยากรมากขึ้น |
| **RAG (Retrieval-Augmented Generation)** | เทคนิคการค้นหาข้อมูลจากฐานความรู้ภายนอก แล้วนำมาประกอบใน Context เพื่อให้โมเดลตอบคำถามโดยใช้ข้อมูลจริง |
| **Planning** | การที่ Agent แตกงานใหญ่ออกเป็นขั้นตอนย่อย วางลำดับ และกำหนดเงื่อนไขความสำเร็จ ก่อนเริ่มลงมือทำ |
| **Reflection** | กระบวนการที่ Agent ตรวจสอบผลลัพธ์ของตัวเอง ประเมินว่าถูกต้องหรือไม่ และปรับปรุงคำตอบก่อนส่งให้ผู้ใช้ |
| **Human-in-the-loop** | การออกแบบให้มนุษย์อยู่ในกระบวนการตัดสินใจที่สำคัญ Agent เสนอแนะ รออนุมัติ หรือขอคำแนะนำเมื่อเจอสถานการณ์ที่ไม่แน่นอน |
| **MCP (Model Context Protocol)** | โปรโตคอลเปิดที่กำหนดวิธีที่ Agent ใช้เชื่อมต่อกับเครื่องมือและแหล่งข้อมูล ทำงานแบบ client-server ช่วยให้ Agent ใช้เครื่องมือต่างๆ ได้ผ่านมาตรฐานเดียวกัน |
| **Multi-agent** | ระบบที่ใช้ Agent หลายตัวทำงานร่วมกัน แต่ละตัวมีบทบาทเฉพาะ เช่น Manager Agent, Coding Agent, Research Agent |
| **Evaluation** | กระบวนการวัดคุณภาพของ Agent ด้วยชุดทดสอบ Metrics เช่น accuracy, recall, task completion rate, cost ต่อ task |
| **Verification** | การตรวจสอบว่าผลลัพธ์ของ Agent ถูกต้อง ตรงตามความต้องการ และไม่เป็นอันตราย |
| **Observability** | ความสามารถในการมองเห็นสถานะภายในของ Agent — ดู logs, traces, ดูว่า Agent คิดอะไร ใช้เครื่องมืออะไร ได้ผลลัพธ์อะไร |
| **Guardrails** | กฎและข้อจำกัดที่วางไว้ให้ Agent ทำงานอยู่ในขอบเขตที่ปลอดภัย เช่น ห้ามลบข้อมูล ห้ามเรียก API บางตัว |
| **Agent Runtime** | ระบบที่ทำหน้าที่รัน Agent จัดการวงจรชีวิต ส่งต่องานระหว่าง Agent กับเครื่องมือ จัดการ memory และ security |
